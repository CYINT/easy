# Easy v0.1.0 Private Beta Release Notes

Status: template, not yet released.

## Access Boundary

This release is a private beta. Access is limited to the approved private network or tunnel for the configured deployment hostname. Do not describe this release as publicly internet reachable unless public HTTPS ingress is verified first.

## Included

- Email/password login with MFA/passkey support.
- Invite-only user onboarding with administrator bootstrap.
- Boards, lists, cards, descriptions, assignments, comments, checklists, attachments, and drag/drop ordering.
- Agent-friendly JSON API under `/api/v1/`.
- API-driven frontend under `/app/`.
- Docker Compose deployment with PostgreSQL.
- Backup, restore, security, and deployment documentation.

## Not Included

- Public internet ingress, unless separately verified before tag creation.
- Google OAuth production release; Google OAuth is disabled by default and has not been manually tested for this release.
- Billing, SaaS tenant management, native mobile apps, plugin marketplace, or workflow automations.

## Release Gates

Before tagging, run:

```powershell
$env:EASY_RELEASE_HOSTNAME="<your-hostname>"
$env:EASY_RELEASE_PRIVATE_BETA_ACCEPTED="true"
$env:EASY_RELEASE_NOTES_PATH="docs/release-notes/v0.1.0-private-beta.md"
npm run release:tag -- v0.1.0 --dry-run
```

Create and push the tag only after the dry run passes and the private-beta access boundary has been accepted by the decision owner.
